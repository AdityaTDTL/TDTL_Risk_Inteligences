# Integration Guide for tdtl‑risk‑ai‑engine

## Overview
The **tdtl‑risk‑ai‑engine** package implements the core AI/ML logic and a FastAPI backend exposing the following service endpoints:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/health` | GET | Health‑check – returns `{"status":"ok"}` |
| `/search-phone-number` | POST | Accepts a phone number (E.164) and returns a detailed `PhoneSearchResponse` JSON (customer profile, transaction summary, fraud analysis, etc.) |
| `/score` | POST | Returns a risk score for a given customer payload (demo placeholder) |
| `/graph` | GET | Provides a stub for future graph‑based collusion data |
| `/agentic` | POST | Demo agentic endpoint (future extension) |

The API is automatically documented at `http://localhost:8000/docs` (Swagger UI) and the OpenAPI spec can be downloaded from `http://localhost:8000/openapi.json`.

## How to Set Up the Backend (for the Backend Team)
1. **Clone the folder** (or receive it as a ZIP) into the desired location.
2. Open a terminal in `tdtl-risk-ai-engine`.
3. Create a virtual environment (recommended) and activate it:
   ```powershell
   python -m venv .venv
   .venv\Scripts\activate   # PowerShell
   # or   .venv\Scripts\activate.bat   # cmd.exe
   ```
4. Install the exact dependencies:
   ```powershell
   pip install -r requirements.txt
   pip install -e .   # editable install of the src package
   ```
5. Run the service locally (ensure the virtual environment is active):
   ```powershell
   # Activate virtual environment if not already active
   .venv\Scripts\activate   # PowerShell
   # Start the FastAPI server
   uvicorn tdtl_risk_ai.api.inference_api:app --host 0.0.0.0 --port 8000
   ```
   - The API becomes reachable at `http://localhost:8000`.
   - Use `Ctrl+C` to stop the server.
   - For development, you may add `--reload` to enable auto‑reloading on code changes.
6. Verify with the health endpoint:
   ```powershell
   curl http://localhost:8000/health
   ```
   Expected output: `{ "status": "ok" }`.
7. Optional – **Dockerize** (useful for the DevOps team):
   ```dockerfile
   # Dockerfile placed at repository root
   FROM python:3.11-slim
   WORKDIR /app
   COPY . /app
   RUN pip install --no-cache-dir -r requirements.txt && \
       pip install -e .
   EXPOSE 8000
   CMD ["uvicorn", "tdtl_risk_ai.api.inference_api:app", "--host", "0.0.0.0", "--port", "8000"]
   ```
   Build & run:
   ```powershell
   docker build -t tdtl-risk-ai .
   docker run -p 8000:8000 tdtl-risk-ai
   ```

## How the Front‑End Team Can Consume the API
1. **Base URL** – the server address where the backend is running, e.g. `http://localhost:8000` (adjust for staging/production).
2. **Swagger UI** – open `http://localhost:8000/docs` to explore request payloads and see example responses.
3. **Sample JavaScript fetch call** (can be dropped into `api_demo.html`):
   ```html
   <script>
   async function searchPhone(phone) {
       const response = await fetch('http://localhost:8000/search-phone-number', {
           method: 'POST',
           headers: { 'Content-Type': 'application/json' },
           body: JSON.stringify({ phone })
       });
       const data = await response.json();
       console.log(data);
       // render `data` in the UI as you prefer
   }

   // Example usage
   searchPhone('+91 9100000001');
   </script>
   ```
4. **Error handling** – the API returns:
   - `404` with `{ "detail": "Customer not found" }` when the phone number is unknown.
   - `422` with a validation error when the phone format is invalid.
   - `200` with the full `PhoneSearchResponse` JSON (see the earlier documentation) on success.
5. **CORS** – the FastAPI app already includes `CORSMiddleware` allowing all origins (`*`). Adjust in `src/tdtl_risk_ai/api/middleware.py` if you need stricter policies.
